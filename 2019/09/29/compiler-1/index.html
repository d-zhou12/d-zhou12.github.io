编译原理 第一章

1.1 语言处理器

1.1.1 编译器和解释器的区别

编译器：将程序设计语言翻译成被计算机执行的形式的软件系统
源程序->编译器->目标程序
解释器：另一种常见的语言处理器；并不通过翻译的方式生成目标程序；直接利用用户输入执行源程序中指定的操作

1.1.2 编译器相对于解释器的优点是什么？解释器相对于编译器的优点是什么？

编译器通常比解释器更快地将用户输入映射为输出，但是解释器错误诊断效果通常比编译器更好，因为它逐个语句地执行源程序

1.1.3 在一个语言处理系统中，编译器产生汇编语言而不是机器语言的好处是什么？

一个语言处理系统： 源程序->预处理器->经过预处理的源程序->编译器->目标汇编程序->汇编器->可重定位机器代码->链接器/加载器->目标机器代码
编译器产生汇编语言的好处就是汇编语言比较容易输出和调试

1.1.4 把一种高级语言翻译成为另一种高级语言的编译器成为源到源的翻译器。编译器使用C语言作为目标语言有什么好处？

正如为什么编译器要把高级语言解释成汇编语言而非机器语言一样，C语言是易于理解和阅读的，它相对汇编比较容易阅读，相对其他高级语言速度快。

1.1.5 描述一下汇编器所要完成的一些任务

处理由编译器产生的汇编语言程序，并生成可重定位的机器码

1.2 一个编译器的结构

编译器的前端和后端

前端：分析部分；把源程序分解成为多个组成要素，并在这些要素之上加上语法结构；然后使用这个结构来创建该源程序的一个中间表示；如果分析查出语法错误或者语义不一致，需要反馈给用户进行改正；还会收集源程序相关信息，存放在符号表中；符号表将和中间表示形式一起传送给综合部分；
后端：综合部分；根据中间表示和符号表中信息来构造用户期待的目标程序

一个编译器的各个步骤

字符流->词法分析器->符号流->语法分析器->语法树->中间代码生成器->中间表示形式->代码生成器->目标机器语言->机器相关代码优化器->目标机器语言

词法分析（lexical analysis/scanning）

将字符流组织成为有意义的词素（lexeme）的序列；对于每个词素，词法分析器产生<token-name，attribute-value>的词法单元（token）作为输出

语法分析（syntax analysis/parsing）

使用词法分析器生成的各个词法单元的第一个分量来创建树形的中间表示。该中间表示给出了词法分析产生的词法单元流的语法结构；一个常用的表示方法是语法树，树的每个内部节点表示一个运算，而该节点的子结点表示该运算的分量；输出即为语法树

语义分析（semantic analyzer）

使用语法树和符号表中的信息来检查源程序是否和语言定义的语义一致；类型检查，自动类型转换

中间代码生成

在语法分析和语义分析完成之后，很多编译器生成一个明确的低级的或者累机器语言的中间表示；如三地址代码，这种中间表示由一组类似于汇编语言的指令组成，每个指令具有三个运算分量；需要具有易于生成，且能够呗轻松地翻译成为目标及其上的语言

符号表管理

记录源程序中使用的变量的名字，收集每个名字的各种属性有关的信息。这些属性可以提供一个名字的存储分配、类型、作用域等信息；对于过程名字：它的参数数量和类型、每个参数的传递方法（如值传递或者传递引用）以及返回类型

程序语言分类

第一代语言

机器语言

第二代语言

汇编语言

高级程序设计语言

Fortran、Cobol、Lisp、C、C++、C#

第四代语言 为特定应用设计的语言

NOMAD、SQL、Postscript

第五代语言 基于逻辑和约束的语言

Prolog、OPS5

强制式和声明式

强制式：指明要进行哪些计算，如C、C++、Java等；所有强制式语言都有用于表示程序和语句的表示方法
声明式：函数式语言ML、Haskell以及约束逻辑语言Prolog等

冯诺依曼语言

以冯诺依曼计算机体系结构为计算模型的程序设计语言

面向对象语言

支持面向对象编程的语言；指一组相互作用的对象组成程序的编程风格

脚本语言

具有高层次运算符的解释型语言，通常用于把多个计算过程粘合在一起。这些计算过程被称为脚本。如Awk、JavaScript、Perl、PHP、Python、Ruby、Tcl等

1.6 程序设计语言基础

静态和动态的区别

静态：如果一个语言使用的策略支持编译器静态决定某个问题，那么就说是用了一个静态策略，或者说这个问题可以在编译时刻决定；
动态：如果一个只允许在运行程序时候做出的决定的策略被称为动态策略，或者说呗认为需要在运行时刻作出决定
作用域：程序的一个区域，其中对x的使用都指向这个声明
静态作用域：如果仅通过阅读程序就可以确定一个声明的作用域，那么这个语言使用的就是静态作用域（词法作用域）；否则就是动态作用域；大部分语言都是用静态作用域
Java中的static：指的不是变量的作用域，而是编译器确定用于存放被声明变量的内存位置的能力

环境与状态

环境：从名字到存储位置的映射
状态：从内存位置到value的映射

函数和方法

函数：一般指面向过程的语言中的函数
方法：一般指面向对象语言中的方法

声明和定义

声明：告诉我们事物的类型
定义：告诉我们它的值

动态作用域

动态作用域对于多态过程是必不可少的；多态过程是指对于同一个名字根据参数类型具有两个或者多个定义的过程

块结构

允许语句块互相嵌套的语言称为块结构的语言

参数传递机制

值调用：对实参求值或拷贝，相应的实参本身不会被改变；但是如果传递指针的话就可以改变这个对象本身的值的
引用调用：实参的地址作为相应的形参的值被传递给被调用者；沿着这个指针找到调用者指明的内存地址，改变形参看起来就像是改变了实参一样；但是如果实参是一个表达式，那么在调用之前首先会对表达式求值，然后它的值被存放在一个该值自己的位置上，改变形参会改变这个位置上的值，但是不会改变调用者的数据

别名

当参数被以引用传递方式传递时，两个形参可能会指向同一个对象；这会造成一个变量的修改改变了另一个变量的值
